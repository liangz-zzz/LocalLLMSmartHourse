# 2D 户型编辑与设备坐标需求

版本：v0.2（2026-07-16）

## 目标
- 支持用户上传 2D 户型图（PNG/JPG），在同一页面编辑房间和设备位置。
- 通过“两点 + 实际距离（米）”建立图片像素与真实长度的统一比例尺。
- 设备布点后自动计算米制 `x/y/z`，并持久化到统一设备模型的 `placement.coordinates`。
- 数据仅在本机持久化，不上传第三方，不进入 Git 仓库。

## 功能需求
1) 资产与户型
- 前端仅上传 PNG/JPG；后端保存到本机目录并返回图片宽高。
- 单家庭使用一个 `floorplans.json`，可包含多张楼层/区域户型。
- 提供户型增删改查 API，写入采用原子替换。

2) 比例尺
- 用户在 2D 图中选择两个点并输入真实距离（米）。
- 比例尺使用原图像素尺寸计算，不受浏览器缩放影响。
- 未设置有效比例尺时禁止新增设备。
- 比例尺变化后，当前户型内所有设备的物理坐标自动重算。
- 替换图片会清空比例尺；含设备的户型重新设置比例尺前不能保存。

3) 房间与设备
- 用户可绘制、编辑、命名和删除房间多边形。
- 设备可在 2D 图中添加和拖动，并设置离地高度 `z`（米）。
- 拖动后自动计算所属房间。
- 保存户型后，将 `roomId` 对应的房间名称同步到统一设备模型的 `placement.room`；设备换房或房间重命名后自动更新。
- 用户手工设置的 `placement.room` 优先于户型派生值，后续户型保存不得覆盖手工值。
- 设备的归一化图像位置保存在户型配置，米制坐标保存在设备属性。

4) 坐标约定
- 户型图片左上角为原点，`x` 向右、`y` 向下、`z` 向上。
- `x/y/z` 均使用米。
- 设备坐标包含 `floorplanId`；不同户型坐标不能直接比较距离。
- 户型坐标是已布点设备的坐标事实来源。

5) 场景与状态
- 场景预览继续读取 `/scenes` 与 `/scenes/:id/expanded`。
- 视图模式继续订阅 WebSocket 设备状态；编辑模式可暂停订阅。

6) 兼容性
- 旧户型中的 3D 字段和设备 `rotation/scale` 可读取但不再写回。
- 不再提供 GLB 上传、三点校准或 3D 预览。
- 旧的无比例尺设备布点可查看，但补齐比例尺前不可保存修改。

## 验收
- 添加、拖动设备或修改高度时，属性面板立即显示更新后的米制坐标。
- 保存户型后，`GET /devices/:id` 返回同一坐标。
- 保存户型后，`GET /devices/:id` 的 `placement.room` 返回设备所在房间名称，不再保留 `unknown_room`。
- 普通设备和语音卫星使用相同坐标结构。
- 删除设备或户型后，只清除由户型派生的坐标，其他设备覆盖配置保持不变。
